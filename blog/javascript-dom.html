Tech Explained: JavaScript and the DOM

Published 11 April 2025


JavaScript, the DOM, and how they relate to each other can be intimidating concepts for someone with no technical programming knowledge or background. However, as with most things in life, it only appears hard and daunting from the outside. With a bit of effort to understand and relate, it's not too bad wrapping your brain around it!


An analogy to describe JavaScript and its relationship to HTML and CSS:

If HTML is the timber, the drywall, the floorboards and the general structure of a house, and CSS is the paint job, the wallpaper, the vinyl, the decor and the general design of the building, then JavaScript can be the power tools that put everything together and define the way we interact with construction of HTML/structure and CSS/design of the webpage/house.

In fact, JavaScript can also be thought of the appliances and smart home devices that people interact with in the home!


Explain control flow and loops using an example process from everyday life:

In JavaScript, control flow is the order in which instructions are carried out for interacting with a webpage. Its script usually runs top to bottom, but with control flow statements, you can manipulate the order of JavaScript instructions - loops are a common example of such statements.

Take the process of driving as an everyday life analogy - when you first get in the driver's seat, your control flow could involve indicating if you need to pull out of a parking spot onto the road, or turning on your headlights if it's dark out, or turning on the wipers if it's raining.

As you're driving, you should be looping the process of checking your mirrors, adjusting the wheel to stay in your lane, tapping the pedals to maintain an appropriate speed, and paying attention to potential hazards such as other vehicles or pedestrians.


Describe what the DOM is and an example of how you might interact with it:

The DOM (short of Document Object Model) is a programming user interface that provides a object-oriented representation of a webpage and is used to manipulate web documents. Web browsers construct the DOM when it loads an HTML document, and structures all its elements as 'objects' in a tree-like representation.

The DOM allows you to modify the structure, style and content of a document, and most importantly, facilitates the scripting of user interactions using JavaScript, which allows for changes on a webpage to take place without needing to reload the page.

To access the DOM, you simply need to input a DOM access method wherever you're doing your scripting, (e.g. in a .js file with a code editor, in a browser's DevTools console, etc.). Here are some common examples of DOM access methods:

getElementByClassName getElementByTagName querySelector querySelectorAll

The fundamental concept of using the DOM is simple: you call upon an object of the webpage, then you apply a DOM 'method(s)' to enact the change or manipulation you intend. An example would be to change the text of a button on a webpage after a user has clicked on it - here is a snippet of code you can use to interact with the DOM:

document.getElementByID('button').innerHTML = 'You clicked on me!'

Let's break down the code:

document 👉 calls the overarching parent object, i.e. the HTML document itself. You always start with 'document' object.

.getElementByID() 👉 is the DOM method to single out an object by referencing its assigned HTML ID. In this case, we've selected the HTML element with the ID='button'

.innerHTML 👉 is the DOM method to change the content of the HTML element we've singled out. Along with the syntax of the equal symbol and quotation marks to denote a string, this changes the text on the button to 'You clicked on me!'

NOTE: the solution provided for this example has been over-simplified just to demonstrate interacting with the DOM. In reality, to actually make this work, you will also need to create a function that enacts the code above, and script a DOM method that detects someone clicking the button (either the .addEventListener or .onclick method) and activates the function. But more on this below!


Explain the difference between accessing data from arrays and objects:

Arrays and objects are both vessels in JavaScript for storing information and data. They can both store multiple values and can have data added, removed, or altered later. They are both loopable, can be nested within each other, and can contain more of themselves. However, they differ fundamentally in the nature of data they are designed to contain.

--------

An object is basically, well, an object, that has properties about it. These properties are listed as key-value pairs, and the order they are listed does not matter. You can label/name each key however you like. Here is a syntax example:

let curlyFries = {
  price: 8;
  stock: 19;
  calories: "530kcal";
  health-rating: "5 stars"
}

To access data in an object, 'dot notation' is best practice - for example: console.log(curlyFries.calories)

--------

An array is basically an ordered list. It is a collection of items that are listed in a specific, indexed order. Here is a syntax example:

let customerOrder = ['Iced Mocha', 'Curly Fries', 'Scrambled Eggs', 'Caramel Slice']

To access data in an array, use brackets selecting the index position of the item you are after. Notably, arrays are zero-based, i.e. the first item is 0 and the third item is 2. For example: console.log(customerOrder[3]) prints 'Caramel Slice'.

--------

Another fun difference between objects and arrays is the way you loop them - for objects you use the for...in loop, and for arrays you use the for...of loop (unless looping by index).


Explain what functions are and why they are helpful:

Functions are basically blocks of code that perform a specific task. They are reusable and be called upon multiple times in the script whenever you need them, saving you time from needing to repeat the same code over and over. Here is a syntax example:

function sum(number1, number2) {
  return number1 + number2;
}

In this example, 'sum' is the name of the function (you can name it anything you like), number1 & 2 in the brackets are the parameters being fed into the function (also named however you like), and 'return' is the meat of the function that defines adding those two parameters together and returns a value. To call this function, you can use console.log(sum(5, 12)), which would calculate 5+12 and return 17.

Here's another example using the 'button' context from question 3 above - notice how no parameters are needed here:

function changeButtonText() {
  document.getElementByID('button').innerHTML = 'You clicked on me!';
}

To call this function, you can then script a line of code using an event handler DOM method: either document.getElementById('Button').onclick = changeButtonText or document.getElementById('Button').addEventListener('click', changeButtonText)

In other cases where you simply need to call the function immediately instead of embedding it in another DOM workflow or console logging it, you can simply script the name of the function followed by brackets, for example: displayShoppingList()

Aside from being reusable and saving the need to repeat code, functions are also great because they:

👉 have a modular nature that helps break down complex problems into smaller manageable chunks

👉 have their own encapsulation scope where variables defined within them are isolated from the rest of the script, preventing potential conflicts

👉 are easy to read and understand
